Getting Started with eeoom GEO

Welcome to the getting started guide for the eeoom GEO WordPress plugin.

eeoom GEO helps WordPress websites improve their readiness for AI-powered search, generative engines and answer engines by focusing on structured data, entity clarity, technical accessibility and source-of-trust signals.

This guide explains the basic setup and the first steps after installing the plugin.

What eeoom GEO helps with

The eeoom GEO plugin is designed to support website owners, agencies and SEO professionals who want to make WordPress websites easier for AI systems to understand, verify and reference.

The plugin focuses on areas such as:

* AI search visibility
* Generative Engine Optimization
* structured data analysis
* Schema.org and JSON-LD checks
* entity clarity
* technical crawlability
* source-of-trust signals
* page-level recommendations
* agency-friendly reporting workflows

eeoom GEO does not guarantee mentions, rankings or citations in AI systems. Instead, it helps improve the technical and semantic signals that may support AI search visibility.

Requirements

Before installing eeoom GEO, make sure your website meets the basic requirements.

Recommended environment:

* WordPress 6.0 or newer
* PHP 8.0 or newer
* HTTPS enabled
* public website access for crawlers and validators
* administrator access for initial setup

Some features may require additional services, API access or an active eeoom account depending on the plugin version and enabled modules.

Installation

Install the eeoom GEO plugin like a regular WordPress plugin.

1. Log in to your WordPress admin area.
2. Go to Plugins → Add New.
3. Upload the plugin ZIP file or install it through the available plugin source.
4. Activate the plugin.
5. Open the eeoom GEO menu in the WordPress admin area.

After activation, the plugin will add its own admin area where you can access checks, settings, reports and recommendations.

First setup

After activating the plugin, start with the basic configuration.

Recommended first steps:

1. Open the eeoom GEO dashboard.
2. Review the detected website information.
3. Check whether your homepage can be accessed publicly.
4. Run the first visibility or structured data check.
5. Review critical issues first.
6. Open page-level recommendations for important URLs.
7. Fix technical and structured data issues step by step.

For most websites, the homepage is the best starting point because it usually defines the main organization, website identity and entity signals.

Start with the homepage

The homepage is often the most important page for entity clarity.

A strong homepage setup should usually include clear signals for:

* the organization or business behind the website
* the official website URL
* the main brand name
* a clear description
* sameAs links to official profiles
* structured data using Schema.org
* clean internal linking to important pages
* crawlable content for search engines and AI systems

If your homepage does not clearly define the main entity, AI systems may have a harder time understanding who the website represents.

Check structured data

Structured data is one of the most important technical foundations for AI search visibility.

eeoom GEO can help identify issues such as:

* missing Organization or LocalBusiness schema
* missing WebSite schema
* missing WebPage identifiers
* weak or inconsistent entity descriptions
* duplicate or conflicting JSON-LD entities
* missing sameAs references
* unclear publisher or author relationships
* incomplete page-level structured data

The goal is not to add as much markup as possible. The goal is to provide clear, consistent and useful structured data that matches the visible content of the website.

Understand GEO recommendations

eeoom GEO recommendations are designed to help improve machine readability, trust and entity clarity.

Typical recommendation categories may include:

* critical technical issues
* structured data improvements
* content clarity suggestions
* entity relationship improvements
* crawlability warnings
* missing trust signals
* page-specific AI search readiness improvements

Start with critical issues first. These are usually the problems that can prevent crawlers, validators or AI systems from understanding the page correctly.

Important pages to analyze first

After checking the homepage, continue with the most important public pages.

Recommended priority:

1. Homepage
2. About page
3. Contact page
4. Main service pages
5. Product or offer pages
6. Location pages
7. Author or expert profile pages
8. Important editorial or knowledge pages

For local businesses, location pages and contact information are especially important.

For experts, consultants and agencies, author pages, profile pages and about pages can be important entity signals.

Public pages only

eeoom GEO is designed to analyze public-facing website content.

In most cases, the plugin should focus on:

* published pages
* published posts
* public landing pages
* public service pages
* public location pages

Drafts, private posts, internal admin pages and non-public content should usually not be part of AI search visibility analysis.

Working with agencies and clients

eeoom GEO is built with agency workflows in mind.

Agencies may want clients or editors to access selected visibility checks, scores or reports without giving them full administrator access.

Depending on the plugin version and configuration, access may be separated between:

* sensitive configuration
* structured data settings
* visibility dashboards
* reports
* recommendations
* page-level checks

Administrators should keep sensitive settings restricted, while non-admin users may be allowed to view or use selected reporting features.

Best practices

For best results, keep the following principles in mind.

Use clear entity information
Your website should clearly explain who you are, what you offer and which official profiles belong to you.

Keep structured data consistent
Schema.org markup should match the visible content of the page and should not conflict with other plugins or themes.

Avoid duplicate entity confusion
Use stable identifiers such as homepage-based @id values where appropriate, for example https://example.com/#organization or https://example.com/#website.

Make important pages crawlable
AI systems and search engines need access to public content. Avoid blocking important pages through robots.txt, noindex tags or restrictive server rules unless intentional.

Improve trust signals
Official profiles, clear contact information, author information, company details and reliable references can help support source-of-trust signals.

Review page-level recommendations
AI search visibility is not only a homepage topic. Important service, product, location and expert pages should also be checked individually.

Common first issues

New users often discover issues such as:

* missing Organization schema
* missing WebSite schema
* weak homepage description
* inconsistent brand name
* missing sameAs links
* blocked AI crawlers
* incomplete JSON-LD
* duplicated schema from multiple plugins
* unclear author or publisher relationships
* important pages without structured data

These issues are common and can usually be improved step by step.

Troubleshooting

If something does not work as expected, check the following first:

* Is the plugin activated?
* Are you logged in with the required WordPress role?
* Is the page published and publicly accessible?
* Is the website blocking crawlers?
* Is another SEO or schema plugin creating conflicting JSON-LD?
* Are there PHP errors in the WordPress debug log?
* Is the WordPress REST API available?
* Is caching serving an outdated version of the page?
